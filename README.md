# EduConnect Data Access Simulator

## Overview

This repository contains a simulated data access decision framework for EduConnect Ghana, an ed-tech platform serving over 50,000 students. It demonstrates secure handling of student data in line with **data classification policies**, **data lifecycle management principles**, and **Ghana DPA compliance**.

The simulator includes real-world scenarios for:

- Marketing campaigns requesting student information  
- Analytics partnerships requiring data sharing  
- Account deletion requests under the "right to be forgotten"

The goal is to provide DevOps, data, and compliance teams with a structured decision-making framework that balances operational needs with privacy, security, and legal obligations.

## Objectives

- Apply **data classification rules** for Public, Internal, and Confidential information  
- Follow the **principle of least privilege** in granting access  
- Identify appropriate **data lifecycle stages** (Create, Use, Share, Archive, Destroy)  
- Incorporate **compliance considerations** under Ghana DPA  
- Provide actionable **next steps** for handling each data access request  

## Usage

1. Review the scenarios in [`data_access_simulator.md`](./data_access_simulator.md)  
2. Follow the decision forms to determine access approvals, conditional approvals, or denials  
3. Apply the recommended **action steps** to enforce data security and compliance  
4. Use this repository as a reference or training tool for DevOps, data governance, and compliance teams  

## Conclusion

This simulator provides a practical framework for managing sensitive student data responsibly. By following the guidance in **data_access_simulator.md**, EduConnect Ghana ensures operational efficiency while maintaining privacy, security, and legal compliance.
